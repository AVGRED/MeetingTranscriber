## 一、准备工作（只需做一次）

### 1.1 安装 Git

1. 打开 https://git-scm.com/download/win
2. 下载后一路点「Next」安装，全部用默认选项即可
3. 安装完成后，在桌面右键菜单里会出现 **「Open Git Bash here」**，说明安装成功

### 1.2 配置 SSH 密钥

> 配置好这个，以后 push 就不需要输密码了。

打开 Git Bash（开始菜单搜 Git Bash），**逐条复制粘贴**下面的命令：

```bash
# 1. 设置你的名字和邮箱（换成你自己的）
git config --global user.name "你的姓名拼音"
git config --global user.email "你的邮箱@example.com"

# 2. 生成 SSH 密钥（一路回车，不要设密码）
ssh-keygen -t ed25519 -C "你的邮箱@example.com"

# 3. 显示公钥，复制输出的一整段文字
cat ~/.ssh/id_ed25519.pub
```

4. 打开 https://github.com/settings/keys
5. 点 **「New SSH key」**
6. Title 随便填（比如"公司电脑"），Key 里粘贴刚才复制的那段文字
7. 点 **「Add SSH key」**

验证是否成功：

```bash
ssh -T git@github.com
```

看到 `Hi 你的用户名!` 就说明配置好了。

### 1.3 下载项目到本地

```bash
# 进入到你要放项目的目录，比如桌面
cd ~/Desktop

# 下载项目
git clone git@github.com:AVGRED/MeetingTranscriber.git

# 进入项目目录
cd MeetingTranscriber

# 切换到 dev 分支（日常开发都在这个分支上）
git checkout dev
```

> ✅ 准备工作到此结束，下面进入日常操作。

## 二、日常开发流程（每次改代码都这样）

### 2.1 开始工作前 → 拉取最新代码

> **每次开始写代码之前，先拉一下，确保你拿的是最新版本。**

在项目目录里打开 Git Bash，执行：

```bash
git checkout dev        # 确保在 dev 分支
git pull origin dev     # 拉取同事的最新改动
```

看到 `Already up to date.` 说明已经是最新的了。

### 2.2 改完代码 → 提交

```bash
# 1. 查看你改了哪些文件（可选，心里有数就行）
git status

# 2. 把所有改动加入暂存
git add -A

# 3. 提交，引号里写你这次改了什么
git commit -m "feat: 添加了语音识别功能"

# 留言建议用中文写清楚，比如：
#   "fix: 修复了录音崩溃的问题"
#   "chore: 更新了依赖库版本"
```

### 2.3 上传到 GitHub

```bash
git push origin dev
```

看到类似 `master -> dev` 的输出就说明上传成功了。

## 三、常见问题

### 3.1 推送被拒绝（冲突）

如果 push 时报这个错：

```
! [rejected]  dev -> dev (fetch first)
```

说明同事在你之前推了新代码，产生了冲突。这样做：

```bash
git pull origin dev --rebase    # 先拉最新代码
git push origin dev             # 再推一次
```

如果出现冲突提示 `CONFLICT`，**截图发群里**，不要自己乱改。

### 3.2 忘记自己在哪个分支

```bash
git branch
```

当前分支前面有个 `*` 号，确保是 `* dev` 再操作。

### 3.3 想撤销刚才的修改

```bash
# 撤销某个文件的所有改动（恢复成上次提交的版本）
git checkout -- 文件名

# 撤销所有文件的改动（慎用！）
git checkout -- .
```

## 四、注意事项

| ✅ 要做 | ❌ 不要做 |
|---------|-----------|
| **只往 `dev` 分支推代码** | 不要推 `master` |
| 每次写代码前先 `git pull` | 不要攒一周的代码一次性推 |
| commit 信息用中文写清楚改了什么 | 不要提交 `local.properties`（已配置忽略） |
| 遇到问题截图发群 | 不要自己乱搜命令乱执行 |
